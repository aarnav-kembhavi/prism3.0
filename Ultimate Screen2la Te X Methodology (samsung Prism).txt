Ultimate Screen2LaTeX (SLC) Methodology Using Lightweight Neural Networks
Context and Objective
As discussed in the meeting with Vanishree ma’am, the objective of this work-let is to design a robust, lightweight Screen-to-LaTeX (Screen2LaTeX) system capable of converting screenshots or image-based PDFs captured from mobile phones, laptops, or TVs into portable, compilable LaTeX source code.
The system must: - Handle text, tables, equations, and figures independently using bounding boxes - Operate under CPU and lightweight neural network constraints - Be robust to screen-capture artifacts (moiré, glare, perspective distortion) - Align with Samsung PRISM expectations and a 6-month research timeline
This document consolidates: 1. A detailed literature survey 2. Limitations of existing methodologies 3. A proposed ultimate hybrid methodology combining the best aspects of both prior plans
________________________________________
1. Detailed Literature Survey (Screen2LaTeX & Image-to-LaTeX)
1.1 Core Baseline Systems
•	PDF2LaTeX (Wang & Liu, 2020)
Introduced a deep-learning-based pipeline for converting mathematical PDFs to LaTeX. Acts as the primary reference baseline but assumes clean PDF inputs and server-grade computation.
•	Im2LaTeX / Pix2Tex
Encoder–decoder architectures for mathematical expression recognition. Effective for equations but limited in handling full document structure.
________________________________________
1.2 Layout Analysis and Region Detection
•	PubLayNet, DocLayNet – Datasets for document layout analysis.
•	YOLO-based Document Layout Models (YOLOv8, DocLayout-YOLO, YOLOv10/12)
Demonstrate that CNN-based single-shot detectors can achieve Transformer-level accuracy with far lower latency, making them suitable for edge deployment.
Key insight: Layout detection must precede OCR, not depend on it.
________________________________________
1.3 Text Recognition (OCR)
•	CRNN (Shi et al., 2016) – Classical CNN + RNN pipeline.
•	PaddleOCR / PP-OCRv4 Mobile – Lightweight, multilingual OCR optimized for CPU.
Insight: OCR alone is insufficient; font size, bounding box geometry, and spatial context are required to reconstruct document semantics (headers vs body text).
________________________________________
1.4 Table Structure Recognition (TSR)
•	PP-Structure / SLANet – Separates table structure from text content.
•	TableNet – CNN-based table detection and structure inference.
•	TFLOP (2025) – Introduces a layout-pointer mechanism for merged cells and complex tables.
Insight: Tables must be treated as structure recovery problems, not line-by-line OCR.
________________________________________
1.5 Mathematical Expression Recognition
•	Watch, Attend and Parse (WAP) – Early encoder–decoder MER model.
•	UniMERNet (2024) – Robust to real-world noise.
•	TexTeller (2025) – SOTA accuracy using large synthetic datasets (teacher model).
Insight: Lightweight models achieve high accuracy only when trained via knowledge distillation.
________________________________________
1.6 Screen-Capture–Specific Research
•	Moiré pattern removal via frequency-domain filtering
•	Document dewarping using DocUNet / DocRes
Critical finding: Most document models fail on screen captures because they ignore capture physics.
________________________________________
2. Limitations of Existing Methodologies
2.1 End-to-End Heavy Models (Nougat, GOT-OCR)
•	Extremely high latency
•	Server-grade memory requirements
•	Unsuitable for mobile or laptop CPU execution
2.2 Lightweight OCR-Only Pipelines
•	Assume clean input
•	Lose table topology
•	Flatten all text to default font
•	Fail under glare, moiré, and perspective distortion
2.3 Generic Pipelines
•	Treat screenshots like scanned PDFs
•	No restoration stage
•	Cascading error propagation
________________________________________
3. Proposed Ultimate Hybrid Methodology
This methodology combines: - Robustness-first design (from research-heavy plan) - Engineering feasibility and simplicity (from LISL pipeline)
3.1 System Overview
Input Screenshot / Image PDF
→ Intelligent Image Normalization
→ Hierarchical Layout Analysis (Router)
→ Specialized Lightweight Decoders
→ Semantic Fusion & LaTeX Compiler
→ Output: main.tex + figures/
________________________________________
3.2 Stage 1: Intelligent Image Normalization
Purpose: Convert corrupted screen captures into OCR-friendly representations.
Modules: - Geometric Rectification: Lightweight DocRes-style CNN - Moiré & Glare Suppression: FFT-based frequency filtering + lightweight inpainting
Outcome: Normalized image approximating scanned-PDF quality.
________________________________________
3.3 Stage 2: Layout Analysis (Router)
•	Model: DocLayout-YOLO-Nano (INT8)
•	Input: Normalized image
•	Output: Bounding boxes for:
o	Text
o	Table
o	Equation
o	Figure
Advantages: - OCR-independent - Fast on CPU - Correct spatial ordering (top-left → bottom-right)
________________________________________
3.4 Stage 3: Specialized Content Extraction
A. Text Extraction
•	Model: PP-OCRv4 Mobile
•	Enhancements:
o	Bounding box size → infer headers vs body text
o	Inline math detection via heuristics
•	Output: Structured LaTeX text (\section{}, paragraphs, inline math)
B. Table Extraction
•	Model: SLANet / TFLOP-style TSR
•	Output:
o	Row/column topology
o	Merged cells
•	Conversion: HTML-like structure → LaTeX tabular
C. Equation Recognition
•	Model: Distilled MER model
o	Teacher: TexTeller / UniMERNet
o	Student: Mobile ViT + RNN
•	Output: Pure LaTeX math expressions
D. Figure Handling
•	Method: PIL/OpenCV crop
•	Save as external image
•	Linked via \includegraphics
________________________________________
3.5 Stage 4: Semantic Fusion & LaTeX Compilation
•	Assemble document in correct reading order
•	Automatically inject required LaTeX packages:
o	amsmath, booktabs, graphicx, etc.
•	Output guaranteed to compile
________________________________________
4. Training Strategy
•	No training from scratch
•	Use pretrained lightweight models
•	Fine-tune with:
o	pdf2latex corpus
o	synthetic screen-noise augmentation
•	Apply knowledge distillation for equations

Conclusion
The proposed Ultimate Screen2LaTeX methodology is neither a naive OCR pipeline nor an impractical heavy model. It is a research-backed, lightweight, robustness-first system explicitly designed for real-world screen captures and aligned with Samsung PRISM expectations.
This methodology maximizes correctness, portability, and feasibility under constrained compute environments.
